# ПЛАН ПРОЕКТА: СИСТЕМА TEXT-TO-CAD

## 1. Определение концепции
- Определение целевых типов объектов (детали машин, мебель, примитивы).
- Выбор формата выходных данных (параметрический код или готовая геометрия).
- Анализ требований к точности и сложности моделей.

## 2. Подготовка данных
- Сбор базы соответствий: текстовое описание <-> геометрические параметры.
- Синтетическая генерация данных для редких объектов.
- Очистка и нормализация текстовых запросов (удаление шума, стандартизация мер).

## 3. Разработка логики интерпретации
- Создание модуля обработки естественного языка (анализ геометрии и размеров).
- Построение системы перевода текста в последовательность инженерных команд.
- Логическая валидация: проверка физической возможности создания объекта.

## 4. Геометрическое ядро
- Реализация механизмов генерации твердых тел.
- Настройка инструментов объединения, вычитания и пересечения фигур.
- Экспорт моделей во внешние форматы для проверки в стороннем ПО.

## 5. Интеграция и интерфейс
- Разработка среды для ввода промптов и визуализации результата.
- Создание системы обратной связи (правка модели через уточняющий текст).
- Оптимизация скорости обработки запросов.

## 6. Тестирование и контроль качества
- Сравнение сгенерированных моделей с эталонными чертежами.
- Проверка корректности топологии (отсутствие дыр, пересечений).
- Стресс-тесты на сложных и противоречивых описаниях.

## 7. Документация и релиз
- Написание руководства пользователя по составлению промптов.
- Описание ограничений системы.
- Финальная сборка проекта для публикации.
